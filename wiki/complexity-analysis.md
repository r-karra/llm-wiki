# Complexity Analysis

**Summary**: The process of determining the amount of time and space resources required to execute an algorithm.

**Sources**: [lesson_1_binary_search_and_complexity_analysis.ipynb](../raw/lesson_1_binary_search_and_complexity_analysis.ipynb)

---

Complexity analysis helps us understand how an algorithm scales as the input size increases.

## Big O Notation
- **O(1)**: Constant time.
- **O(log n)**: Logarithmic time (e.g., [[binary-search]]).
- **O(n)**: Linear time.
- **O(n log n)**: Linearithmic time (e.g., merge sort).
- **O(n^2)**: Quadratic time.

## Types
- **Time Complexity**: Execution time.
- **Space Complexity**: Memory usage.
